## How to back the last commit

`--hard`: Resets the working directory and staging area to match the specified commit. Discards all changes.

```bash
git reset --hard HEAD~1
```

`--soft`: Moves HEAD to the specified commit but keeps changes staged. Useful for amending commits.
```bash
git reset --soft HEAD~1
```

`--mixed`: (default): Resets the staging area but keeps working directory changes. Unstages changes from the last commit.
```bash
git reset --mixed HEAD~1
```

You can use the same command to back multiple commits.

```bash
git reset --hard HEAD~3
git reset --soft HEAD~3
git reset --mixed HEAD~3
```

## How to restore files and delete untracked files

Restore all files to the last commit.
```bash
git checkout .
```

Delete untracked files.
```bash
git clean -fdx
```

## How to setup email, name and editor

```bash
git config --global user.email "your@email.com"
git config --global user.name "Your Name"
git config --global core.editor "code --wait"
```

## How to show, create, renaming and deleting branches

```bash
git checkout -b branch-name # create and switch to branch
git branch -m old-branch-name new-branch-name # rename branch
git branch -d branch-name # delete branch
git branch -a # list all branches
```
## How to rename the last commit

`--amend`: That flag allows you to change the last commit message.

```bash
git commit --amend -m "new commit message"
```
